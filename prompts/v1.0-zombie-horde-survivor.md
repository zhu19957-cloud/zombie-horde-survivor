Create a complete playable browser-based top-down auto-attack survivor game inspired by zombie horde shooters that runs by opening a single index.html file and features massive enemy waves, a 3-choice level-up upgrade system, out-of-game permanent progression, 3 distinct playable characters with unique abilities and ultimates, stage-ending boss fights, and a gold-based economy.

==================================================
GAME CONCEPT
==================================================

A top-down auto-attack survivor where the player controls a character that automatically attacks enemies while the player focuses on positioning and upgrade choices. The core fantasy is a power trip: start vulnerable, become a one-army wrecking ball mowing down hundreds of enemies. Each stage escalates from manageable swarms to screen-filling hordes, culminating in a boss fight. Gold collected during runs persists for permanent out-of-game upgrades, creating a satisfying progression loop across sessions.

ARCHITECTURE & LAYOUT
==================================================

**Screen States** (navigated via a simple state machine):
1. Title Screen — game logo, "Start Game" button
2. Character Select — 3 character cards with name, portrait icon, stat preview, ability description
3. Out-of-Game Shop — gold balance display, grid of permanent upgrades with level and cost
4. Stage Select — unlocked stages listed, each showing recommended power level

**In-Game HUD Layout:**

- Top-right: Gold count, level + XP bar
- Bottom-center: Ultimate ability icon with cooldown timer

- Floating text: damage numbers, gold pickup indicators, level-up notification

**Level-Up Overlay:**
- Semi-transparent dark overlay pauses the game
- 3 upgrade cards arranged horizontally in the center
- Each card: upgrade name, icon, description, rarity border color
- Click a card to select it and resume the game
- Cards scale up on hover for clear feedback

CORE SYSTEMS
==================================================

**Character System**

Three playable characters, each with a distinct auto-attack pattern, passive trait, and ultimate ability:

1. **Guardian** — melee tank
   - Auto-attack: Cleave — swings weapon in a 120-degree arc in front, damages all enemies within range (cooldown: 0.8s, range: 60px, damage: base ATK)
   - Passive: Fortify — gain 3% damage reduction per enemy within 100px (max 30%)
   - Ultimate: Fortress — become invulnerable for 5 seconds, emit a shockwave every 1 second that deals 50 damage and knocks back all enemies within 150px (cooldown: 60s)
   - Base stats: HP 120, ATK 12, SPD 2.0, ATKSPD 1.25, CRIT 5%, CRITDMG 150%

2. **Gunslinger** — ranged DPS
   - Auto-attack: Shoot — fires a bullet at the nearest enemy (cooldown: 0.5s, range: 250px, damage: base ATK, projectile speed: 8px/frame)
   - Passive: Sharpshooter — every 5th attack is a guaranteed crit with 2x knockback
   - Ultimate: Bullet Hell — for 8 seconds, fire 5 bullets simultaneously in a spread pattern every 0.2s, each bullet pierces through 3 enemies (cooldown: 60s)
   - Base stats: HP 70, ATK 10, SPD 2.5, ATKSPD 2.0, CRIT 10%, CRITDMG 200%

3. **Shadow** — fast assassin
   - Auto-attack: Dash Strike — blink to the nearest enemy within range, deal damage, then blink back (cooldown: 0.6s, range: 150px, damage: base ATK × 1.2)
   - Passive: Afterimage — each attack leaves a shadow clone that auto-attacks the same target once for 40% damage after 0.3s
   - Ultimate: Time Warp — slow all enemies to 30% speed for 6 seconds, gain 2x attack speed, spawn 2 afterimage clones that independently attack nearby enemies (cooldown: 60s)
   - Base stats: HP 80, ATK 9, SPD 3.5, ATKSPD 1.67, CRIT 20%, CRITDMG 180%

**Auto-Attack Behavior:**
- Characters automatically target the nearest enemy within attack range
- Guardian faces the direction of the targeted enemy and swings
- Gunslinger fires a projectile that travels toward the target position
- Shadow blinks to the target, deals damage, then returns to original position
- If no enemy is in range, the character does nothing until one enters range
- Auto-attack cooldown is modified by ATKSPD stat

**Enemy System**

Enemies spawn continuously from outside the viewport edges and move toward the player. Spawn rate and enemy variety increase with stage time.

Normal enemies (spawn in massive quantities):

1. Walker — HP 10, Speed 1.0, Damage 5, Size 12px, Color #4a7c59. Walks directly toward player. The bread-and-butter enemy.
2. Runner — HP 7, Speed 2.5, Damage 5, Size 10px, Color #7c4a4a. Fast but fragile. Spawns in bursts of 5-8.
3. Swarmer — HP 5, Speed 2.0, Damage 3, Size 8px, Color #6b6b4a. Tiny and spawns in clusters of 10-15. Individually weak but overwhelming in numbers.

Special enemies (spawn occasionally starting from wave 3, have distinct behaviors):

4. Spitter — HP 20, Speed 0.8, Damage 8 (projectile), Size 14px, Color #4a4a7c. Stops at 120px from player and fires a projectile every 2s. Projectile speed: 4px/frame.
5. Bomber — HP 15, Speed 1.5, Damage 15 (AoE), Size 13px, Color #7c5a4a. Walks toward player. On death, explodes dealing 15 damage in 60px radius after a 0.5s delay. Visual warning: pulses red for 0.5s before explosion.
6. Shielder — HP 25, Speed 1.0, Damage 5, Size 15px, Color #5a5a5a. Takes 50% reduced damage from attacks hitting its front arc (facing direction). Takes full damage from attacks hitting its back arc. Always faces the player.
7. Healer — HP 18, Speed 0.8, Damage 3, Size 12px, Color #4a7c7c. Heals all enemies within 80px for 3 HP every 2 seconds. Emits a green pulse ring when healing.

Boss enemies (one per stage, spawns at the 4:00 mark):

8. Stage 1 Boss — Colossus: HP 2000, Speed 0.6, Damage 20, Size 40px, Color #3d2b1f. Ground slam every 5s: damages all players within 100px for 20 damage and knocks them back. Summons 5 Walkers every 10s. Visual: screen shakes on slam.
9. Stage 2 Boss — Necromancer: HP 1500, Speed 1.0, Damage 12, Size 30px, Color #2b1f3d. Fires 3 dark orbs every 3s (damage 12 each, speed 3px/frame). Teleports to a random position when hit for more than 50 damage in a single instance. Summons 1 Spitter every 8s and 1 Shielder every 12s.
10. Stage 3 Boss — Hive Queen: HP 3000, Speed 0.8, Damage 15, Size 45px, Color #3d3d1f. Charges across the screen every 6s (damage 25 on contact, leaves acid trail for 3s dealing 5 damage/s). Spawns 8 Swarmers every 5s. Phase 2 at 50% HP: charge frequency doubles, spawns 2 Spitters every 8s in addition to Swarmers.

**Spawn System:**
- Enemies spawn outside the viewport edges at random positions
- Base spawn rate: 1 enemy per 0.5s at stage start, increasing to 1 per 0.15s by 3 minutes, and 1 per 0.08s at 4 minutes (boss spawn)
- Normal enemy ratio: Walker 50%, Runner 30%, Swarmer 20% initially; Swarmer ratio increases over time
- Special enemies: first special spawns at 0:45, then every 15-25s randomly
- Spawn rate scales with a global multiplier that increases every 30 seconds: ×1.0 → ×1.2 → ×1.5 → ×1.8 → ×2.2 → ×2.8
- No hard cap on enemy count — use spatial hashing and culling to maintain performance

**In-Game Upgrade System**

On each level-up, the game pauses and displays 3 randomly selected upgrades. The player picks 1. Each upgrade appears as a card with name, icon, and description.

Upgrade pool (grouped by category, each with weight for random selection):

Offensive (weight: 40%):
- Damage Up: ATK +15%
- Attack Speed Up: ATKSPD +15%
- Crit Chance Up: CRIT +8%
- Crit Damage Up: CRITDMG +25%
- Projectile Count +1 (Gunslinger only): fire 1 additional bullet per shot
- Piercing Shot (Gunslinger only): bullets pierce through 1 additional enemy
- Cleave Arc +30° (Guardian only): increase arc width of cleave
- Cleave Range +20% (Guardian only): increase reach of cleave
- Dash Range +25% (Shadow only): increase dash strike range
- Afterimage Damage +25% (Shadow only): increase shadow clone damage

Defensive (weight: 30%):
- Max HP Up: Max HP +20, heal 20 HP
- Armor Up: reduce all incoming damage by 3 (minimum 1)
- HP Regen: recover 1 HP per second
- Shield: gain a shield that absorbs 1 hit every 15 seconds
- Dodge Chance: 8% chance to avoid any damage instance

Utility (weight: 30%):
- Move Speed Up: SPD +12%
- Pickup Range Up: increase XP orb and gold pickup radius by 30%
- Gold Bonus: gold earned +20%
- XP Bonus: XP earned +20%
- Cooldown Reduction: ultimate cooldown -15%

Rarity and scaling:
- Each upgrade can appear multiple times (stacking). Diminishing returns: each stack of the same upgrade provides 80% of the previous stack's value.
- At levels 5, 10, and 15, one of the 3 cards is guaranteed to be a "rare" version with 2× the effect (golden border).
- Maximum 20 upgrades per run (level cap 20).

**Out-of-Game Shop System**

Permanent upgrades purchased with gold. Gold persists across runs via localStorage. Each upgrade has multiple levels with escalating costs.

| Upgrade | Per Level | Max Level | Cost Formula |
|---|---|---|---|
| Vitality | Max HP +5 | 20 | 50 × level |
| Power | ATK +2 | 20 | 60 × level |
| Agility | SPD +0.2 | 10 | 80 × level |
| Precision | CRIT +2% | 10 | 100 × level |
| Greed | Gold multiplier +10% | 10 | 200 × level |
| Head Start | Start at level +1 | 5 | 300 × level |

All 3 characters share the same permanent upgrade pool. Upgrades apply to the base stats before character-specific modifiers.

**Gold Economy:**
- Walker kill: 1 gold
- Runner kill: 1 gold
- Swarmer kill: 1 gold
- Spitter kill: 3 gold
- Bomber kill: 3 gold
- Shielder kill: 4 gold
- Healer kill: 4 gold
- Boss kill: 50 + (stage number × 25) gold
- Stage clear bonus: 100 × stage number gold
- Gold multiplier from Greed upgrade applies to all gold earned
- Gold is collected automatically when enemies die (within pickup range)
- Gold persists even if the player dies mid-run

**XP and Leveling:**
- All enemies drop 1 XP orb on death (size and value scale with enemy type: normal = 1 XP, special = 5 XP, boss = 50 XP)
- XP orbs are attracted to the player within pickup range and auto-collected on contact
- XP to next level: 10 × (current level + 1)
- Level cap: 20

**Stage System:**
- Stage 1: timer 5:00, boss at 4:00, base difficulty multiplier ×1.0
- Stage 2: timer 5:00, boss at 4:00, base difficulty multiplier ×1.5, special enemies spawn from the start
- Stage 3: timer 5:00, boss at 4:00, base difficulty multiplier ×2.0, special enemies spawn more frequently, double normal spawn rate
- Stages unlock sequentially: Stage 1 is always available, Stage 2 unlocks after clearing Stage 1, Stage 3 unlocks after clearing Stage 2
- Clear condition: kill the boss before the timer hits 5:00
- Fail condition: HP reaches 0 (keep gold, return to shop)

**Save System:**
- Save to localStorage on every: gold change, shop purchase, stage unlock
- Save data structure: { gold: number, upgrades: { vitality: number, power: number, agility: number, precision: number, greed: number, headStart: number }, unlockedStages: number[], totalRuns: number, totalKills: number }
- Load save data on game start; if no save exists, initialize with defaults

TECHNICAL IMPLEMENTATION
==================================================

**Game Loop:**
- requestAnimationFrame with delta time calculation
- Update order: input → movement → auto-attack → projectiles → enemy AI → collisions → particles → HUD → render
- Pause game loop when in level-up, pause menu, or non-gameplay screens

**Performance with Massive Enemy Counts:**
- Spatial hash grid for collision detection: divide world into 64×64px cells, each cell contains references to enemies in that region. Check collisions only against enemies in the same or adjacent cells as the player/projectile.
- Viewport culling: only render enemies and particles within the visible canvas area plus a 50px margin
- Object pooling for projectiles, particles, and XP orbs: pre-allocate arrays, reuse objects instead of creating/destroying
- Batch rendering: group draw calls by color/type, minimize context state changes
- Enemy count soft limit: if enemies exceed 500, stop spawning new normal enemies until count drops below 450. Special enemies and bosses ignore this limit.

**Collision Detection:**
- Circle-circle collision for all entities (player, enemies, projectiles, orbs)
- Collision radius = size/2 for each entity
- Player auto-attack hit detection: arc check for Guardian (angle + range), line check for Gunslinger (projectile vs enemy), distance check for Shadow (dash target)

**Canvas Rendering Order (back to front):**
1. Ground / grid pattern
2. XP orbs
3. Enemies (sorted by y-position for slight depth)
4. Projectiles
5. Player character
6. Particles
7. HUD overlay
8. Boss HP bar (if boss active)
9. Level-up overlay (if leveling up)

**Input Handling:**
- Track key states in a Set for continuous movement (keydown adds, keyup removes)
- Mouse position tracked for UI interactions only
- Prevent default on arrow keys and Space to avoid page scrolling

**Save/Load:**
- Serialize game state to JSON string, store in localStorage key "zombieHordeSurvivor"
- Load on startup, validate structure, migrate if fields missing
- Save after every: gold change, shop purchase, stage clear
