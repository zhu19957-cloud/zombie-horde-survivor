==================================================
GAME CONCEPT
==================================================

A top-down auto-attack survivor where the player controls a character that automatically attacks enemies while the player focuses on positioning and upgrade choices. The core fantasy is a power trip: start vulnerable, become a one-army wrecking ball mowing down hundreds of enemies. Each stage escalates from manageable swarms to screen-filling hordes, culminating in a boss fight. Gold collected during runs persists for permanent out-of-game upgrades, creating a satisfying progression loop across sessions. The game supports English and Chinese with a toggle accessible from the title screen and pause menu.

**Character System**

Three playable characters, each with a distinct auto-attack pattern, passive trait, and ultimate ability:

1. **Guardian** — melee tank
   - Auto-attack: Cleave — swings weapon in a 120-degree arc in front, damages all enemies within range (cooldown: 0.8s, range: 60px, damage: base ATK)
   - Passive: Fortify — gain 3% damage reduction per enemy within 100px (max 30%)
   - Ultimate: Fortress — become invulnerable for 3 seconds, emit a shockwave every 1.5 seconds that deals 25 damage and knocks back all enemies within 120px (cooldown: 90s)
   - Base stats: HP 120, ATK 12, SPD 2.0, ATKSPD 1.25, CRIT 5%, CRITDMG 150%

2. **Gunslinger** — ranged DPS
   - Auto-attack: Shoot — fires a bullet at the nearest enemy (cooldown: 0.5s, range: 250px, damage: base ATK, projectile speed: 8px/frame)
   - Passive: Sharpshooter — every 5th attack is a guaranteed crit with 2x knockback
   - Ultimate: Bullet Hell — for 5 seconds, fire 3 bullets simultaneously in a spread pattern every 0.35s, each bullet pierces through 2 enemies (cooldown: 90s)
   - Base stats: HP 70, ATK 10, SPD 2.5, ATKSPD 2.0, CRIT 10%, CRITDMG 200%

3. **Shadow** — fast assassin
   - Auto-attack: Dash Strike — blink to the nearest enemy within range, deal damage, then blink back (cooldown: 0.6s, range: 150px, damage: base ATK × 1.2)
   - Passive: Afterimage — each attack leaves a shadow clone that auto-attacks the same target once for 40% damage after 0.3s
   - Ultimate: Time Warp — slow all enemies to 50% speed for 5 seconds, gain 1.5x attack speed, spawn 1 afterimage clone that independently attacks nearby enemies (cooldown: 90s)
   - Base stats: HP 80, ATK 9, SPD 3.5, ATKSPD 1.67, CRIT 20%, CRITDMG 180%

**Auto-Attack Behavior:**
- Characters automatically target the nearest enemy within attack range every frame
- Each frame, find the enemy with the shortest distance to the player that is within the character's attack range
- If an enemy is in range and the auto-attack cooldown has elapsed, execute the attack immediately — do not wait for the next frame after cooldown expires
- Guardian faces the direction of the targeted enemy and swings
- Gunslinger fires a projectile that travels toward the target position
- Shadow blinks to the target, deals damage, then returns to original position
- If no enemy is in range, the character does nothing until one enters range
- Auto-attack cooldown is modified by ATKSPD stat
- Auto-attack must function independently of player movement — the character attacks while moving
- Auto-attack targeting must use the same coordinate space as enemy positions — verify with a simple distance calculation each frame

**Spawn System:**
- Enemies spawn outside the viewport edges
- Distribute spawn positions around the viewport perimeter: divide the perimeter into 8 equal segments. Each spawn cycle picks 2-4 random segments and spawns enemies at positions within those segments, ensuring no two spawn points are within 60px of each other in the same cycle
- This prevents all enemies from entering from the same direction and clustering into a single wall
- New enemies fade in over 0.3s (alpha 0 → 1) to prevent visual pop-in and give the player a moment to react
- Base spawn rate: 1 enemy per 0.5s at stage start, increasing to 1 per 0.15s by 3 minutes, and 1 per 0.08s at 4 minutes (boss spawn)
- Normal enemy ratio: Walker 50%, Runner 30%, Swarmer 20% initially; Swarmer ratio increases over time
- Special enemies: first special spawns at 0:45, then every 15-25s randomly
- Spawn rate scales with a global multiplier that increases every 30 seconds: ×1.0 → ×1.2 → ×1.5 → ×1.8 → ×2.2 → ×2.8
- No hard cap on enemy count — use spatial hashing and culling to maintain performance

**Collision and Separation System:**
- Player and enemies have solid collision volumes — they cannot pass through each other
- When the player and an enemy overlap, push both apart along the axis connecting their centers. Player receives 70% of the push, enemy receives 30% (player is heavier). The push strength is enough to fully separate them within 2 frames.
- Enemies cannot push the player by walking into them — the collision resolution prevents this. The player is never displaced by enemy movement alone; only by boss knockback abilities or enemy collision resolution.
- Enemies have collision with each other: when two enemies overlap, push both apart equally along the axis connecting their centers. Separation force is gentle — enemies can compress slightly when surrounded by many others, but should never fully overlap (minimum gap: 3px between enemy edges).
- Use the spatial hash grid for efficient neighbor lookup during collision resolution.
- Process collision resolution after enemy movement each frame, before rendering.
- Performance optimization: for enemy-enemy separation, only check enemies in the same or adjacent spatial hash cells. Limit to 3 iterations per frame to prevent cascading resolution from tanking performance.

TECHNICAL IMPLEMENTATION
==================================================

**Game Loop:**
- requestAnimationFrame with delta time calculation
- Update order: input → movement → collision resolution (player-enemy separation, enemy-enemy separation) → auto-attack targeting → auto-attack execution → projectiles → enemy AI → collisions (damage) → particles → HUD → render
- Pause game loop when in level-up, pause menu, or non-gameplay screens

**Auto-Attack Implementation:**
- Each frame: decrement autoAttackCooldown by deltaTime
- When autoAttackCooldown reaches 0: find the nearest enemy within attack range using distance calculation (Math.hypot). If found, execute the attack and reset autoAttackCooldown to 1 / ATKSPD seconds
- Auto-attack targeting uses the same coordinate system as enemy positions — no offset, no scaling
- Auto-attack fires immediately when cooldown reaches 0 and a target is in range — no additional delay
- Auto-attack state (autoAttackCooldown, autoAttackTarget) is independent of player movement input

**Performance with Massive Enemy Counts:**
- Spatial hash grid for collision detection: divide world into 64×64px cells, each cell contains references to enemies in that region. Check collisions only against enemies in the same or adjacent cells as the player/projectile.
- Viewport culling: only render enemies and particles within the visible canvas area plus a 50px margin
- Object pooling for projectiles, particles, and XP orbs: pre-allocate arrays, reuse objects instead of creating/destroying
- Batch rendering: group draw calls by color/type, minimize context state changes
- Enemy count soft limit: if enemies exceed 500, stop spawning new normal enemies until count drops below 450. Special enemies and bosses ignore this limit.

**Collision Detection and Resolution:**
- Circle-circle collision for all entities (player, enemies, projectiles, orbs)
- Collision radius = size/2 for each entity
- Player auto-attack hit detection: arc check for Guardian (angle + range), line check for Gunslinger (projectile vs enemy), distance check for Shadow (dash target)
- Player-enemy separation: when player and enemy circles overlap, calculate overlap distance. Push player 70% and enemy 30% of the overlap distance along the axis connecting their centers. Apply immediately, no accumulation.
- Enemy-enemy separation: when two enemy circles overlap, push each enemy 50% of the overlap distance along the axis connecting their centers. Maintain minimum 3px gap between edges. Limit separation resolution to 3 iterations per frame to cap performance cost. Use spatial hash grid for neighbor lookup.
- Enemies cannot push the player by walking into them — the separation resolution prevents this. Enemy movement moves the enemy toward the player, but collision resolution pushes them apart if they overlap.
- The player cannot walk through enemies — if the player moves into an enemy, collision resolution pushes them apart.

**Spawn Distribution:**
- Calculate the viewport perimeter and divide into 8 equal segments (top-left edge, top edge, top-right edge, right edge, bottom-right edge, bottom edge, bottom-left edge, left edge)
- Each spawn cycle selects 2-4 random segments
- Within each selected segment, pick a random position along that edge
- Enforce minimum 60px distance between spawn positions within the same cycle
- This distributes enemies around the player instead of clustering them in one direction

**Canvas Rendering Order (back to front):**
1. Ground / grid pattern
2. Frozen Trail segments (if active)
3. XP orbs
4. Enemies (sorted by y-position for slight depth)
5. Projectiles
6. Player character
7. Orbital Shield orbs (if active)
8. Particles
9. HUD overlay
10. Boss HP bar (if boss active)
11. Level-up overlay (if leveling up)

**Input Handling:**
- Track key states in a Set for continuous movement (keydown adds, keyup removes)
- Mouse position tracked for UI interactions only
- Prevent default on arrow keys and Space to avoid page scrolling

**Language Implementation:**
- Define a `translations` object at the top of the script containing all UI strings for both `en` and `zh` keys
- Example structure: `const translations = { en: { startGame: "Start Game", hp: "HP", gold: "Gold", ... }, zh: { startGame: "开始游戏", hp: "生命值", gold: "金币", ... } }`
- All text rendering reads from `translations[currentLanguage]`
- Switching language updates `currentLanguage` and re-renders all visible text immediately
- Save language preference to localStorage on change

**Save/Load:**
- Serialize game state to JSON string, store in localStorage key "zombieHordeSurvivor"
- Load on startup, validate structure, migrate if fields missing
- Save after every: gold change, shop purchase, stage clear, language change
